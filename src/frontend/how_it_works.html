Comment fonctionne MEGAM ARG Detection ?

Une explication simple de l'architecture de l'application, de votre clic jusqu'aux résultats.

Architecture de l'application

L'application suit une architecture classique en 3 couches :

🖥️

Frontend

Interface utilisateur

Les pages web que vous utilisez : formulaires de lancement, tableau de bord de suivi, et pages de resultats. Le navigateur envoie des requetes HTTP a l'API.

HTML + CSS + JavaScript
🔗

API REST

Couche de communication

L'interface entre le frontend et le backend. Recoit les requetes HTTP (POST, GET), les valide, et retourne les reponses au format JSON.

Routes /api/launch, /api/status, /api/results
⚙️

Backend + Pipeline

Moteur d'analyse

Le serveur Python (FastAPI) gere les jobs et lance le pipeline bash qui orchestre les outils bioinformatiques (AMRFinderPlus, ResFinder, CARD/RGI) via Conda.

FastAPI + Pipeline Bash + Conda
Requete HTTP
➡
Validation + Routage
➡
Execution Pipeline
➡
Resultats JSON

Du clic aux resultats : etape par etape

Voici exactement ce qu'il se passe quand vous lancez une analyse :

1

Vous remplissez le formulaire

Vous entrez un identifiant (ex: SRR28083254) dans le formulaire de la page "Nouvelle Analyse". Vous choisissez les options (threads, assembleur...) et cliquez sur "Lancer l'analyse".

Frontend : form_launch_analysis.html
2

Le navigateur envoie une requete HTTP

Votre navigateur envoie une requete POST a l'API. C'est comme envoyer une lettre avec votre commande dedans.

// Le navigateur envoie ceci :
POST /api/launch

{
  "sample_id": "SRR28083254",
  "threads": 4,
  "assembler": "megahit"
}
Frontend (api-client.js) ➡ API (FastAPI)
3

Le backend cree le job et lance le pipeline

FastAPI recoit la requete, cree une entree dans la base de donnees SQLite (status: PENDING), puis lance le pipeline bash en arriere-plan. Le status passe a RUNNING.

# Le backend fait :
1. INSERT INTO jobs (...) -- cree le job en DB
2. subprocess.Popen(pipeline.sh) -- lance le pipeline
3. return {"job_id": "abc123"} -- repond au frontend
Backend : main.py + pipeline_launcher.py + database.py
4

Le monitoring en temps reel (polling)

Le frontend interroge regulierement l'API (polling toutes les 3 secondes) pour connaitre l'etat d'avancement du job : statut, etape en cours, et progression estimee.

// Toutes les 5 secondes :
GET /api/status/abc123

// Reponse :
{
  "status": "RUNNING",
  "current_step": "AMRFinderPlus",
  "progress": "60%"
}
Frontend (polling) ⇄ Backend (parsing logs)
5

Le pipeline bash fait le vrai travail

Le script bash orchestre les outils bioinformatiques les uns apres les autres, chacun dans son environnement conda :

📥
Telechargement
SRA / GenBank
🔧
Assemblage
SPAdes / MEGAHIT
🧬
Annotation
Prokka
💊
AMRFinderPlus
Detection ARG
🔬
ResFinder
Detection ARG
🃏
CARD / RGI
Detection ARG
Pipeline : MANUAL_MEGA_MONOLITHIC_PIPELINE_v3.2_WEB.sh
6

Les resultats s'affichent

Le pipeline est termine ! Le backend parse les fichiers de sortie (TSV, HTML) et les renvoie au frontend sous forme de JSON. Le status passe a COMPLETED.

GET /api/results/abc123

{
  "amrfinder": [{ "gene": "blaTEM-1", "resistance": "ampicillin" }],
  "resfinder": [...],
  "card_rgi": [...]
}
Backend (output_parser.py) ➡ Frontend (page_results_arg.html)

Glossaire : les termes techniques en simple

Pas de panique, voici la traduction :

Terme technique En simple
Frontend La partie visible de l'application : les pages, boutons, formulaires que vous utilisez.
Backend La partie invisible qui fait les calculs, stocke les donnees et lance les analyses.
API Le "langage" que le frontend et le backend utilisent pour communiquer (des requetes HTTP avec du JSON).
HTTP / HTTPS Le protocole de communication du web. Comme le systeme postal, mais pour les donnees.
GET / POST GET = demander des informations. POST = envoyer des informations.
JSON Un format texte structure pour echanger des donnees. Comme un formulaire papier, mais pour les machines.
Nginx Le portier de l'application : il recoit toutes les requetes et les aiguille au bon endroit.
Uvicorn Le moteur qui fait tourner FastAPI. Il ecoute les requetes API et les traite.
FastAPI Le framework Python qui definit les routes API (/api/launch, /api/results...).
Docker Une "boite" qui contient toute l'application et ses dependances, pour que ca marche partout pareil.
Conda Un gestionnaire d'environnements qui installe les outils bioinformatiques avec les bonnes versions.
SQLite La base de donnees (un simple fichier) qui stocke la liste des analyses et leur statut.
Polling Demander regulierement "c'est pret ?" au backend. Comme un enfant a l'arriere de la voiture : "On est arrives ?"
Pipeline Une chaine d'outils qui s'executent les uns apres les autres, comme une chaine de montage.
ARG Antimicrobial Resistance Gene — gene de resistance aux antimicrobiens. C'est ce que le pipeline detecte.

En resume

Vous cliquez sur un bouton (frontend), le navigateur envoie un message (API), Python lance les outils bioinformatiques (backend + pipeline), et les resultats s'affichent dans votre navigateur.

C'est tout !
