Du clic aux résultats : étape par étape

Voici exactement ce qu'il se passe quand vous lancez une analyse :

1

Vous remplissez le formulaire

Vous entrez un identifiant (ex: SRR28083254) ou uploadez vos fichiers FASTQ/FASTA dans le formulaire "Nouvelle Analyse". Vous choisissez les options et cliquez sur "Lancer l'analyse".

Frontend : form_launch_analysis.html
2

Le navigateur envoie une requête HTTP

Votre navigateur envoie une requête POST à l'API avec les paramètres d'analyse.

// Le navigateur envoie ceci :
POST /api/launch

{
  "sample_id": "SRR28083254",
  "threads": 4,
  "prokka_mode": "auto"
}
Frontend (api-client.js) ➡ API (FastAPI)
3

Le backend crée le job et lance le pipeline

FastAPI reçoit la requête, crée une entrée dans la base de données SQLite (status: PENDING), puis lance le pipeline bash en arrière-plan. Le status passe à RUNNING.

# Le backend fait :
1. INSERT INTO jobs (...) -- crée le job en DB
2. subprocess.Popen(pipeline.sh) -- lance le pipeline
3. return {"job_id": "abc123"} -- répond au frontend
Backend : main.py + pipeline_launcher.py + database.py
4

Le monitoring en temps réel (polling)

Le frontend interroge régulièrement l'API (polling toutes les 3-5 secondes) pour connaître l'état d'avancement du job : statut, étape en cours, et progression estimée.

// Toutes les 5 secondes :
GET /api/status/abc123

// Réponse :
{
  "status": "RUNNING",
  "current_step": "AMRFinderPlus",
  "progress": "60%"
}
Frontend (polling) ⇄ Backend (parsing logs)
5

Le pipeline bash fait le vrai travail

Le script bash orchestre les outils bioinformatiques les uns après les autres, chacun dans son environnement Mamba :

Contrôle Qualité

FastQC

Ce que fait cet outil

FastQC analyse la qualité brute des reads de séquençage. Il évalue la distribution des scores de qualité par base, le contenu en GC, la présence d'adaptateurs, les séquences surreprésentées et les niveaux de duplication.

Son rôle dans le pipeline

Il intervient en tout premier pour vérifier que les données brutes sont exploitables avant tout traitement. C'est le diagnostic initial qui détermine si un nettoyage sera nécessaire.

Comment interpréter les résultats

Les modules passés en vert sont satisfaisants. Les avertissements orange méritent attention mais ne sont pas bloquants. Les échecs rouges (adaptateurs, qualité basse en fin de read) sont corrigés par fastp à l'étape suivante.

fastp

Ce que fait cet outil

fastp effectue le trimming et le filtrage des reads : suppression des adaptateurs, élimination des reads de mauvaise qualité, correction des bases et filtrage par longueur minimale.

Son rôle dans le pipeline

Il nettoie les données brutes pour que l'assemblage en aval soit de meilleure qualité. Des reads propres produisent des contigs plus longs et plus fiables.

Comment interpréter les résultats

Vérifiez le pourcentage de reads conservés (idéalement > 85%). Le rapport indique le nombre de reads filtrés et la raison (qualité, longueur, adaptateurs). Un taux de filtrage élevé peut indiquer un problème de séquençage.

MultiQC

Ce que fait cet outil

MultiQC agrège les rapports de tous les outils QC (FastQC, fastp, QUAST, etc.) en un seul rapport HTML interactif avec des graphiques comparatifs.

Son rôle dans le pipeline

Il fournit une vue d'ensemble unifiée de toutes les métriques de qualité. Au lieu de consulter chaque rapport séparément, vous avez tout en un coup d'œil.

Comment interpréter les résultats

Le rapport interactif permet de comparer les échantillons entre eux. Cherchez les outliers dans les graphiques : un échantillon très différent des autres mérite une investigation.

Acquisition des données

sra-tools

Ce que fait cet outil

sra-tools (fasterq-dump, prefetch) télécharge les données de séquençage depuis la base NCBI SRA. Il convertit les fichiers SRA en fichiers FASTQ utilisables par le pipeline.

Son rôle dans le pipeline

C'est le point d'entrée quand vous analysez des données publiques via un numéro d'accession SRA (ex: SRR1234567). Il récupère automatiquement les reads paired-end ou single-end.

Comment interpréter les résultats

Le téléchargement produit des fichiers FASTQ (_1.fastq et _2.fastq pour du paired-end). Vérifiez que les fichiers ne sont pas vides et que le nombre de reads correspond aux métadonnées SRA.

Assemblage

SPAdes

Ce que fait cet outil

SPAdes est un assembleur de novo qui reconstruit le génome à partir des reads courts. Il utilise des graphes de Bruijn avec plusieurs valeurs de k-mer pour produire des contigs et scaffolds.

Son rôle dans le pipeline

L'assemblage est l'étape centrale : il transforme des millions de reads courts en séquences génomiques continues. Sans assemblage, impossible d'annoter les gènes ou de détecter les gènes de résistance sur des contigs complets.

Comment interpréter les résultats

Un bon assemblage a un N50 élevé (idéalement > 50 kb pour une bactérie), peu de contigs (< 200) et une taille totale cohérente avec le génome attendu. Vérifiez aussi que le coverage moyen est suffisant (> 30x).

seqkit

Ce que fait cet outil

seqkit est un outil de manipulation de séquences FASTA/FASTQ. Il calcule des statistiques (nombre de séquences, longueur, N50) et filtre les contigs par taille minimale.

Son rôle dans le pipeline

Après l'assemblage, seqkit filtre les contigs trop courts (< 500 bp) qui sont souvent du bruit, et fournit des statistiques de base sur l'assemblage.

Comment interpréter les résultats

Les statistiques seqkit montrent le nombre total de contigs, la longueur totale, le N50 et la longueur du plus long contig. Comparez la taille totale avec la taille attendue du génome de votre organisme.

Évaluation de l'assemblage

QUAST

Ce que fait cet outil

QUAST évalue la qualité de l'assemblage en calculant des métriques détaillées : N50, L50, nombre de contigs, taille totale, plus long contig, contenu GC, et nombre de N par 100 kbp.

Son rôle dans le pipeline

QUAST valide que l'assemblage est de qualité suffisante pour les analyses en aval. Un mauvais assemblage produirait des résultats de détection de résistance peu fiables.

Comment interpréter les résultats

Un N50 > 50 kb est bon pour une bactérie. Le nombre de contigs < 200 est acceptable. La taille totale doit correspondre à la taille attendue du génome (+/- 10%). Un GC% aberrant peut indiquer une contamination.

Annotation

Prokka

Ce que fait cet outil

Prokka annote rapidement les génomes procaryotes : il prédit les gènes codants (CDS), les ARN ribosomiques (rRNA), les ARN de transfert (tRNA) et les éléments régulateurs. Il attribue des noms de gènes et des fonctions par homologie.

Son rôle dans le pipeline

L'annotation transforme des séquences brutes en informations biologiques exploitables. Elle identifie chaque gène et sa fonction potentielle, ce qui est essentiel pour comprendre le contenu génétique de l'organisme.

Comment interpréter les résultats

Le nombre de CDS doit être cohérent avec la taille du génome (environ 1 gène pour 1 kb). Les protéines hypothétiques sont normales (30-40% typiquement). Les fichiers GFF et GBK peuvent être visualisés dans des outils comme Artemis ou IGV.

Typage moléculaire

MLST

Ce que fait cet outil

MLST (Multi-Locus Sequence Typing) détermine le Sequence Type (ST) d'un isolat en analysant les allèles de 7 gènes de ménage conservés. Chaque combinaison unique d'allèles définit un ST.

Son rôle dans le pipeline

Le typage MLST permet de situer l'isolat dans un contexte épidémiologique. Certains STs sont associés à des clones épidémiques connus (ex: ST131 chez E. coli, ST258 chez K. pneumoniae) ou à des profils de résistance spécifiques.

Comment interpréter les résultats

Le schéma identifié correspond à l'espèce détectée. Le ST est un numéro unique. Si le ST est "Non détecté", cela signifie que la combinaison d'allèles ne correspond à aucun ST connu dans la base PubMLST, mais le profil allélique reste informatif.

Détection des gènes de résistance (ARG)

AMRFinderPlus

Ce que fait cet outil

AMRFinderPlus (NCBI) identifie les gènes de résistance aux antimicrobiens, les gènes de virulence et les gènes de tolérance au stress. Il utilise une base de données curée par le NCBI avec des modèles HMM et BLAST.

Son rôle dans le pipeline

C'est l'outil de référence du NCBI pour la détection ARG. Sa base est régulièrement mise à jour et inclut la nomenclature standardisée. Il détecte aussi les mutations ponctuelles de résistance chez certaines espèces.

Comment interpréter les résultats

Chaque gène détecté indique la classe d'antibiotique concernée et le pourcentage d'identité/couverture. Une identité > 90% avec une couverture > 80% est un résultat fiable.

ResFinder

Ce que fait cet outil

ResFinder (Center for Genomic Epidemiology) détecte les gènes de résistance acquise aux antimicrobiens par alignement BLAST contre sa propre base de données curée.

Son rôle dans le pipeline

ResFinder complète AMRFinderPlus avec une base de données indépendante. Le croisement de deux sources améliore la sensibilité et la spécificité de la détection.

Comment interpréter les résultats

Les résultats montrent le gène, le phénotype prédit, l'identité et la couverture. Les seuils par défaut sont 90% identité et 60% couverture. Les gènes avec une couverture partielle peuvent être des fragments.

CARD / RGI

Ce que fait cet outil

RGI (Resistance Gene Identifier) utilise la base CARD (Comprehensive Antibiotic Resistance Database) avec des modèles de détection par homologie protéique, variants SNP et modèles de sur-expression.

Son rôle dans le pipeline

CARD/RGI apporte une troisième perspective indépendante et classifie les mécanismes de résistance (efflux, modification de cible, inactivation enzymatique).

Comment interpréter les résultats

Les résultats "Perfect" indiquent une correspondance exacte avec la base CARD. "Strict" indique une détection fiable. "Loose" sont des hits moins fiables à interpréter avec prudence.

Virulence et plasmides

VFDB

Ce que fait cet outil

VFDB (Virulence Factors Database) identifie les facteurs de virulence bactériens : adhésines, toxines, systèmes de sécrétion, capsules, sidérophores et autres déterminants de pathogénicité.

Son rôle dans le pipeline

La détection de virulence complète l'analyse de résistance. Un isolat résistant ET virulent représente une menace plus importante.

Comment interpréter les résultats

Les facteurs de virulence sont classés par catégorie (adhésion, toxine, etc.). Comparez avec les profils de virulence typiques de l'espèce pour identifier des acquisitions inhabituelles.

PlasmidFinder

Ce que fait cet outil

PlasmidFinder identifie les réplicons plasmidiques dans les assemblages. Il détecte les groupes d'incompatibilité (IncF, IncI, IncN, etc.) qui classifient les plasmides selon leur mécanisme de réplication.

Son rôle dans le pipeline

Les plasmides sont les principaux vecteurs de dissémination des gènes de résistance. Identifier les plasmides présents aide à évaluer le risque de transfert horizontal.

Comment interpréter les résultats

Chaque réplicon détecté indique un type de plasmide. Certains groupes d'incompatibilité sont connus pour porter des gènes de résistance spécifiques (ex: IncF avec blaCTX-M).

Détection de mutations

PointFinder

Ce que fait cet outil

PointFinder détecte les mutations ponctuelles (SNPs) connues pour conférer une résistance aux antibiotiques. Il cible les gènes chromosomiques comme gyrA, parC, rpoB, etc.

Son rôle dans le pipeline

Les mutations ponctuelles sont sur le chromosome. Elles sont particulièrement importantes pour les fluoroquinolones, la rifampicine et la colistine.

Comment interpréter les résultats

Chaque mutation est décrite par le gène, la position et le changement d'acide aminé (ex: gyrA S83L). Les mutations "connues" dans la base sont les plus fiables.

Snippy

Ce que fait cet outil

Snippy détecte les variants (SNPs et indels) en alignant les reads sur un génome de référence. Il produit un fichier VCF annoté avec l'impact de chaque variant sur les gènes.

Son rôle dans le pipeline

Snippy fournit une analyse complète des variants au-delà des mutations de résistance connues, permettant de détecter des mutations nouvelles ou rares.

Comment interpréter les résultats

Les variants sont classés par impact : HIGH (frameshift, stop gained), MODERATE (missense), LOW (synonyme). Concentrez-vous sur les variants HIGH et MODERATE dans les gènes connus de résistance.

En résumé

Le pipeline utilise 15 outils complémentaires qui couvrent toute la chaîne : de la qualité des reads bruts jusqu'à la détection des gènes de résistance, mutations, virulence et plasmides.

Chaque outil apporte un angle différent. C'est leur combinaison qui donne des résultats fiables.
